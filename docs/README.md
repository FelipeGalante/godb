# GoDB documentation

This directory holds everything that isn't code:

- **[Product Requirements Document](prd.md)** — what GoDB is, who it's for, what v0.1 needs to do, what is deliberately out of scope.
- **[Architecture Decision Records](adr/)** — the load-bearing decisions made along the way, each with the context that forced them, the alternatives considered, and the consequences accepted.
- **[The development book](book/)** — a chapter-per-milestone narrative explaining the database-internals concepts behind each layer, then walking through the code. Reads from the start of the project to the current milestone.

If you are reading the repo cold, start with [the book introduction](book/00-introduction.md) and follow chapters in order. The PRD answers "what is being built"; the ADRs answer "why the design looks like this"; the book answers "how would I learn to build this myself."
